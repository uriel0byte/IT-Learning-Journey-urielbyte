## 🎯 Featured Case Studies

### [Day 3: Splunk SIEM Log Analysis - Did you SIEM?](./04-Case-Studies/case-study-splunk-log-analysis.md) ⭐
Investigated ransomware attack using Splunk to analyze web traffic and firewall logs. Traced complete attack chain from reconnaissance through SQL injection to data exfiltration. Identified C2 communication and calculated 126KB of stolen data.

**Skills:** SPL (Search Processing Language), log correlation, anomaly detection, attack chain reconstruction, C2 detection

**Key Takeaways:** Complete SPL syntax reference, filtering best practices, investigation workflow templates

---

### [Day 10: SOC Alert Triaging - Tinsel Triage](./04-Case-Studies/case-study-soc-alert-triage.md) ⭐⭐⭐
**Most Important for SOC Career** - Simulated real SOC analyst workflow using Microsoft Sentinel. Triaged 8 incidents (4 high-severity privilege escalation alerts) across 4 Linux hosts. Used KQL to correlate alerts, reconstruct attack chains, and identify kernel module persistence.

**Skills:** SIEM operations, 4-factor triage model (Severity, Time, Stage, Impact), KQL query writing, cloud security (Azure), multi-host incident correlation, privilege escalation detection

**Key Takeaways:** Complete triage methodology, KQL templates, attack chain patterns, SOC daily workflow

---

### [Day 6: Malware Analysis - Egg-xecutable](./04-Case-Studies/case-study-malware-analysis.md) ⭐
Analyzed HopHelper.exe using static (PeStudio) and dynamic (ProcMon, Regshot) techniques. Identified registry persistence mechanism, extracted C2 IP address from strings, and confirmed TCP-based command-and-control communication.

**Skills:** Static malware analysis, dynamic analysis, sandbox methodology, IOC extraction, persistence detection

**Key Takeaways:** Complete malware analysis workflow, Sysmon detection rules, ProcMon filtering guide

---

### [Day 7: Network Discovery with Nmap](./04-Case-Studies/case-study-network-discovery.md)
Performed comprehensive network reconnaissance to regain access to compromised server. Discovered FTP on non-standard port 21212, custom TBFC app on 25251, and localhost-only MySQL database. Demonstrated importance of full port scanning and service enumeration.

**Skills:** Network scanning (TCP/UDP), service enumeration, protocol analysis, on-host discovery

**Key Takeaways:** Complete Nmap command reference, port vs. protocol concepts, detection signatures

As an ILS student transitioning to cybersecurity, I needed practical, hands-on experience to complement my Security+ theoretical studies. AOC 2025 provided structured daily challenges that built progressively from foundational to advanced concepts.

### What I Learned
- **Technical Skills:** SIEM operations (Splunk, Sentinel), malware analysis (static/dynamic), network scanning, web exploitation, container security, cloud security (Azure, AWS), ICS/OT security
- **Problem-Solving:** Systematic investigation methodologies, attack chain analysis, multi-source log correlation

- **Tool Proficiency:** 50+ industry-standard tools including Splunk, Nmap, Burp Suite, Azure Sentinel, CyberChef, AWS CLI, RITA, Zeek, Registry Explorer

### Growth & Challenges
The event started with accessible Linux basics and gradually increased in complexity. Days 3 (Splunk), 10 (Azure Sentinel), 15 (Web Forensics), 19 (ICS/SCADA), and 22 (C2 Detection) were particularly challenging, requiring deeper understanding of log correlation, cloud security, industrial systems, and network-based threat hunting.

- Some days (web security, programming-heavy challenges) were outside my comfort zone

- Persistence and documentation proved more valuable than natural aptitude
- Enhanced documentation serves as personal reference for future SOC work

### Connection to Career Goals

This practical experience complements my Security+ preparation and demonstrates to recruiters that I can apply theoretical knowledge in real scenarios.
